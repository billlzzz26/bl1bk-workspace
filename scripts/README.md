# 🛠️ Development Cleanup Tools

ชุดเครื่องมือสำหรับค้นหาและทำความสะอาดไฟล์ขยะจากการพัฒนาโปรเจกต์ (node_modules, target) เพื่อคืนพื้นที่จัดเก็บข้อมูลบน Android (Termux)

## 📁 รายชื่อสคริปต์

| ไฟล์ | หน้าที่ |
|----------|----------|
| `find_dev_dirs.sh` | 🔍 ค้นหาโฟลเดอร์ `.git`, `node_modules`, และ `target` ทั่วเครื่อง และอัปเดตไฟล์คอนฟิก |
| `cleanup_node.sh` | 🧹 ลบโฟลเดอร์ `node_modules` ตามรายการในคอนฟิก (มีการยืนยัน y/n) |
| `cleanup_target.sh` | 🦀 ลบโฟลเดอร์ `target` (Rust Build) ตามรายการในคอนฟิก (มีการยืนยัน y/n) |

## 🚀 วิธีใช้งาน

### 1. การเตรียมการ (ครั้งแรก)
ให้สิทธิ์การรันกับสคริปต์ทั้งหมดในโฟลเดอร์นี้:
```bash
chmod +x scripts/*.sh
```

### 2. ค้นหาไฟล์ขยะล่าสุด
รันสคริปต์ค้นหาเพื่อสแกนเครื่องและบันทึกรายชื่อโฟลเดอร์ที่พบลงในไฟล์ `.json`:
```bash
./scripts/find_dev_dirs.sh
```
*สคริปต์นี้จะสร้างไฟล์ `remove_node.config.json` และ `remove_target.config.json` มาให้โดยอัตโนมัติ*

### 3. เริ่มการทำความสะอาด
เลือกสั่งลบตามประเภทที่ต้องการ ระบบจะแสดงชื่อโปรเจกต์และถามยืนยัน **y/n** ทุกครั้งก่อนลบ:

**ลบ node_modules:**
```bash
./scripts/cleanup_node.sh
```

**ลบ Rust target (Build artifacts):**
```bash
./scripts/cleanup_target.sh
```

## ⚠️ ข้อควรระวัง
- **การลบเป็นการลบแบบถาวร (`rm -rf`)** ไม่สามารถกู้คืนจากถังขยะได้
- โปรดอ่านชื่อโปรเจกต์ที่ปรากฏในคำถามยืนยันให้ดีก่อนตอบ `y`
- สคริปต์เหล่านี้ออกแบบมาเพื่อรันบน Termux โดยสแกนพื้นที่ `/storage/emulated/0` เป็นหลัก
